## Root Cause Analysis: Global Hotkey
The hotkey appeared to work (indicator showed recording state) but recordings were silently lost because:
1. Main window is hidden when user presses hotkey (closed to tray)
2. `requestAnimationFrame` doesn't fire in hidden windows → energy tracker never collected samples
3. `getAudioStats()` returned `speechDetected: false` (zero samples)
4. Recording was discarded with "No speech detected" — but user never saw this message because window was hidden
5. Additionally, `getUserMedia()` takes 500ms+ and stop could arrive before start finished, causing the stop to be silently ignored

The indicator window showed everything working because it receives commands independently from the main process and doesn't do any audio processing.
